Backlog · estimated

US-1: Check-In in CureMD + Blank Note (ERP → CureMD) Appointments

As front-desk / clinical staff I want check-in from ERP to drive CureMD, blank note job, and sleep/x-ray orders so that we avoid double entry.
What This Does: When staff check a patient in through the ERP system, it automatically updates the patient's status in CureMD and creates a blank provider note ready for the doctor to use. No more forgetting to manually create notes in CureMD.
Why This Matters

This is the most important user story in the backlog. Front-desk staff are consistently forgetting to manually create the provider note in CureMD when a patient checks in — resulting in downstream workflow failures where the doctor arrives at the exam room and has no note to document in. The original MVP deferred this functionality entirely due to time pressure. The client has been doing it manually since launch and the error rate has not improved. The proposed flow reverses the current workflow: instead of the user working in CureMD to check in the patient and then separately creating the note, the user checks the patient in from the ERP platform, which triggers a chain — (1) the ERP syncs the check-in status to CureMD via API, (2) a blank provider note is auto-created in CureMD as part of that chain, and (3) the ERP handles both check-in and check-out so the user never needs to touch CureMD for this step.

US-2: Inbound CureMD Check-In Sync + Poll Appointments

As staff using CureMD for check-in I want ERP updated within minutes so that modalities stay aligned.
What This Does: When operations team checks a patient in at CureMD (their established workflow), the ERP automatically detects the check-in and creates the corresponding order in ClinicSync MD. No more double-entry.
Why This Matters

This is the reverse direction of US-1 — where US-1 flows ERP → CureMD, this story flows CureMD → ERP. The client's operations team currently checks patients in at CureMD (this is their established workflow and they are not going to stop doing it), and what they want is for that check-in event to automatically trigger the creation of the corresponding order in ClinicSync MD. Today, staff must manually create orders in the ERP after checking in at CureMD, which is redundant double-entry. The CureMD API provides live access to the scheduler, meaning the ERP can poll the appointment list and pick up status changes (including check-ins) in near-real-time. This eliminates the old FEDS-style approach where a report had to be exported, downloaded, and parsed — the connection is now live and bidirectional.

Approver confirms admin sign-in from phone; admin completes session after approve.

Provides frictionless login that preserves identity control. Additional security layer discussed: same-IP/location detection so re-auth isn't needed from the same network. Feature applies only to specific user types (doctors), not all users.
