User story

As operations or clinical leadership,
I want to export a CSV report from each modality module,
So that I can review all orders and all statuses without manual copy-paste from the UI.

Requirement

Add an Export CSV control on each module’s orders/report surface:

  • X-Ray
  • Sleep Study
  • PFT
  • PST

Each export must include all orders and all statuses for that module (full module dataset — not limited to the current on-screen filter unless product explicitly chooses filter-scoped export later).

Solution outline

LayerDeliverable
Backend Shared CSV serialization helper · one authenticated export route per module (four routes) · reuse existing list/query patterns · consistent column naming where possible across modalities
Frontend Shared Export CSV button component on four module pages · loading/disabled state · browser file download · same label and placement pattern on each page
QA Full regression on one module (row count, status coverage, permissions) · smoke on the other three modules

Acceptance criteria

  • Each of the four modules exposes a working Export CSV action visible to authorized users.
  • Downloaded file includes all orders for that module and all workflow statuses represented in the ERP.
  • Export respects tenant and user access rules (same as list views).
  • Button UX is consistent across modules (label, placement, loading feedback).
  • No regression to existing list/filter behaviour when export is not used.
